Introduction

Feature selection is an essential step in machine learning, particularly in regression modeling. Selecting the right subset of features can significantly enhance model performance, reduce overfitting, and improve interpretability. However, traditional methods such as stepwise regression and LASSO often fail in high-dimensional or complex datasets.

This project applies Genetic Algorithms (GAs), an evolutionary approach inspired by natural selection, to optimize feature selection for regression modeling. By simulating the process of biological evolution, GAs enable us to explore vast combinations of features efficiently and find the most relevant ones.

Why Genetic Algorithms?

Traditional feature selection techniques rely on deterministic rules that may overlook nonlinear interactions between features. Genetic Algorithms, on the other hand, use mutation, crossover, and selection to find optimal feature subsets adaptively.

This makes GAs an excellent choice for feature selection in cases where: - The dataset is high-dimensional. - Features exhibit nonlinear relationships. - We want to balance interpretability with predictive power.

Methodology

1. Chromosome Encoding

Each potential feature subset is represented as a chromosome, a binary vector where: - 1 â†’ Feature is included - 0 â†’ Feature is excluded

For example, a chromosome like:

c(1, 0, 1, 1, 0, 0, 1) # Features 1, 3, 4, and 7 selected

represents a model that uses features 1, 3, 4, and 7.

2. Evolutionary Process

The Genetic Algorithm follows these key steps:

  • Initialization: Generate a random population of feature subsets.
  • Selection: Choose the best-performing feature sets based on a fitness function.
  • Crossover: Combine feature subsets from two parent solutions.
  • Mutation: Randomly flip 0 and 1 values to introduce diversity.
  • Elitism: Preserve the best feature sets across generations.

The goal is to minimize the mean squared error (MSE) while reducing the number of selected features.

Dataset & Preprocessing

Sonar Dataset: Mines vs. Rocks

The dataset consists of 208 sonar signals used to distinguish between: - Mines (M): Metal cylinders. - Rocks (R): Naturally occurring structures.

Each observation contains 60 frequency band readings, making this a high-dimensional classification problem.

library(tidyverse)
library(caret)

# Load dataset
sonar <- read.csv("./data/sonar.csv")

# Convert target variable to factor
sonar$Class <- factor(sonar$Class, levels = c("R", "M"))

# Train-test split
set.seed(123)
trainIndex <- createDataPartition(sonar$Class, p = 0.7, list = FALSE)
trainData <- sonar[trainIndex, ]
testData  <- sonar[-trainIndex, ]

Feature Selection Using Genetic Algorithm

Defining the Fitness Function

The GA optimizes feature selection by evaluating subsets based on their predictive accuracy. We define our fitness function using a k-Nearest Neighbors (k-NN) classifier, where the fitness score is calculated as:

\[ Fitness = 1 - Accuracy \]

library(GA)

fitness_function <- function(features) {
  selected_features <- names(trainData)[features == 1]
  model <- train(Class ~ ., data = trainData[, c(selected_features, "Class")],
                 method = "knn", trControl = trainControl(method = "cv", number = 5))
  return(1 - max(model$results$Accuracy))
}

# Run Genetic Algorithm
set.seed(2025)
ga_model <- ga(type = "binary", fitness = fitness_function,
               nBits = ncol(trainData) - 1, popSize = 50,
               maxiter = 100, elitism = 5, mutationRate = 0.01)
